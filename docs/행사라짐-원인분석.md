# 행이 사라지는 현상 - 원인 코드 분석

## 1. 핵심 원인: `renderCalendar()` 전체 재렌더

**Calendar.js 2509-2517**
```javascript
function renderCalendar() {
  document.querySelectorAll(".calendar-1day-drag-drop-line").forEach((el) => el.remove());
  document.querySelectorAll(".calendar-1day-resize-preview-line").forEach((el) => el.remove());
  const grid = getCalendarGridFor1Day(dayOffset);
  // ...
  calendarGrid.innerHTML = "";  // ← 전체 DOM 비움
  // ...
  renderTimeBudgetTablesForCalendar(budgetColumn, targetKey, todoSection);  // ← 새로 생성
  calendarGrid.appendChild(budgetColumn);
  // ...
}
```

`renderCalendar()`가 호출될 때마다 **전체 1일 뷰 DOM이 비워지고 새로 생성**됨.  
→ 편집 중이던 행이 DOM에서 제거되고, `storedGoals`/`tasksFromToday` 기준으로 다시 그려짐.

## 2. `renderCalendar()`를 호출하는 경로

### 2-1. `calendar-budget-scheduled-updated` 이벤트 (주요 원인)

**Calendar.js 2961-2963**
```javascript
document.addEventListener("calendar-budget-scheduled-updated", function onBudgetScheduledUpdated() {
  if (document.contains(wrap)) renderCalendar();
});
```

**이 이벤트를 발생시키는 곳:**

| 파일 | 줄 | 상황 |
|------|-----|------|
| Time.js | 5556-5560 | `saveCurrentGoal(skipReRender=false)` 호출 시 |
| Time.js | 5603-5606 | 행 삭제 버튼 클릭 시 |

→ `saveCurrentGoal`에서 `skipReRender=true`로 바꿔 이벤트를 막은 상태.

---

### 2-2. "오늘의 할일" 테이블 blur

**Calendar.js 2608-2626**
```javascript
const saveScheduled = () => {
  const start = startInput.value.trim();
  const end = endInput.value.trim();
  // ... updateKpiTodo 등 호출 ...
  if (ok) {
    t.startTime = start;
    t.endTime = end;
    renderCalendar();   // ← 여기서 전체 재렌더
    refreshTodoList();
  }
};
startInput.addEventListener("blur", saveScheduled);
endInput.addEventListener("blur", saveScheduled);
```

투자/소비 내역 입력란을 클릭할 때, **이전에 포커스가 있던 요소가 "오늘의 할일"의 start/end 입력이면**  
→ blur → `saveScheduled` → `ok`일 때 `renderCalendar()` 호출 → 전체 재렌더.

---

### 2-3. "오늘의 할일" 체크박스 클릭

**Calendar.js 2576-2591**
```javascript
bar.addEventListener("click", (e) => {
  e.stopPropagation();
  const newDone = !t.done;
  // ... syncKpiTodoCompleted 등 ...
  t.done = newDone;
  bar.classList.toggle("is-completed", newDone);
  bar.querySelector(".calendar-monthly-span-bar-checkbox-inner")?.classList.toggle("checked", newDone);
  renderCalendar();   // ← 여기서 전체 재렌더
  refreshTodoList();
});
```

할일 체크박스를 클릭하면 항상 `renderCalendar()` 호출.

---

### 2-4. createTagDropdown의 document click

**Time.js 1059-1061**
```javascript
document.addEventListener("click", (e) => {
  if (!wrap.contains(e.target)) panel.hidden = true;
});
```

드롭다운 패널만 닫고, `renderCalendar`는 호출하지 않음.

## 3. 요약: 행이 사라지는 흐름

1. 사용자가 투자/소비 내역에서 "과제 기록" → 새 행 추가
2. 과제명/예상시간 등 입력란 클릭
3. 그 전에 포커스가 "오늘의 할일" 입력에 있었거나, 할일 체크박스를 클릭함
4. `renderCalendar()` 호출 → 전체 DOM 재생성
5. 재생성 시 `storedGoals`에 저장된 행만 다시 그림
6. **저장되지 않은 행** 또는 **플레이스홀더가 제대로 저장되지 않은 행**은 사라짐

---

## 4. 수정 방향

1. **이미 적용된 것**
   - `saveCurrentGoal`에서 `skipReRender=true`로 이벤트 미발송
   - 새 행 추가 시 플레이스홀더로 즉시 저장

2. **추가로 고려할 것**
   - "오늘의 할일" blur 시 `renderCalendar()` 호출 최소화  
     (예: 값이 실제로 변경된 경우에만 호출)
   - 할일 체크박스 클릭 시 전체 재렌더 대신, 해당 행만 업데이트하는 방식 검토
