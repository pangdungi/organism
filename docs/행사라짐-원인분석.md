# 행이 사라지는 현상 - 원인 코드 분석 및 해결

## 1. 핵심 원인

**`renderCalendar()` 전체 재렌더**가 budget 테이블(투자/소비 내역)까지 포함해 DOM을 비우고 새로 생성함.

```javascript
// Calendar.js - renderCalendar()
calendarGrid.innerHTML = "";  // ← 전체 DOM 비움
// ...
renderTimeBudgetTablesForCalendar(budgetColumn, targetKey, todoSection);  // ← 새로 생성
```

→ 편집 중이던 행이 DOM에서 제거되고, `storedGoals`/`tasksFromToday` 기준으로 다시 그려짐.

## 3. `calendar-budget-scheduled-updated` 이벤트 발생 경로

| 파일 | 줄 | 상황 |
|------|-----|------|
| Time.js | ~5571 | 예상 시작/마감 blur 시 (조건부) |
| Time.js | ~5624 | 행 삭제 버튼 클릭 시 |

**Time.js 방어 로직 (이벤트 미발송 조건):**
- `scheduledTime` 변경 없음
- `blurFromEnter` (Enter 키로 blur)
- `container.contains(activeElement)` (budget 영역 내부 클릭)

---

## 4. 근본 해결 (2024 적용)

**`calendar-budget-scheduled-updated` 수신 시 전체 `renderCalendar()` 대신 timetable 오버레이만 갱신**

```javascript
// Calendar.js - 이벤트 핸들러
if (timeTableInner && dateStr) {
  const { expected, actual } = build1DayTimetableOverlays(dateStr);
  oldExpected?.replaceWith(expected);
  oldActual?.replaceWith(actual);
} else {
  renderCalendar();
}
```

→ budget 테이블은 **재구성하지 않음**. 예상/실제 시간표 오버레이만 교체.

---

## 5. 행이 사라질 수 있는 잔여 시나리오

1. **오늘의 할일 blur** → budget 영역 밖 클릭 시 `renderCalendar()` 호출 (값 변경 시)
2. **nav 버튼** (오늘/이전/다음) → 날짜 변경 시 전체 재렌더
3. **되돌리기** 컨텍스트 메뉴 → 전체 재렌더

→ 1번은 `budgetColumn.contains(activeElement)` 체크로 완화됨.

---

## 6. 요약

- **원인**: 예상 시간 변경 시 `renderCalendar()` 전체 호출 → budget 테이블까지 재구성
- **해결**: `build1DayTimetableOverlays()`로 timetable만 갱신, budget 테이블 유지
- **추가 방어**: blur/Enter/영역 클릭 시 이벤트 미발송 (Time.js)
