# 가장 쉽고 빨리 배포하기

데이터는 **Supabase**, 배포는 **Vercel** 한 번 연결하면 끝입니다.

---

## 1. Supabase (데이터 저장소, 무료)

1. [supabase.com](https://supabase.com) 가입 후 **New project** 생성.
2. 왼쪽 **SQL Editor** 들어가서 아래 SQL **Run** 한 번 실행.

```sql
-- 테이블 생성
create table if not exists entries (
  id uuid default gen_random_uuid() primary key,
  name text,
  created_at timestamptz default now()
);

-- 익명 사용자도 읽기/쓰기 허용 (나중에 로그인 붙이면 바꾸면 됨)
alter table entries enable row level security;

create policy "Allow all for anon"
  on entries for all
  to anon
  using (true)
  with check (true);
```

3. **Settings → API** 에서 **Project URL** 과 **anon public** key 복사.

4. 프로젝트 폴더에 `.env` 파일 만들고 넣기:

```
VITE_SUPABASE_URL=여기에_Project_URL
VITE_SUPABASE_ANON_KEY=여기에_anon_key
```

5. `npm run dev` 로 실행 → 화면에서 "저장" 눌러보면 Supabase에 저장됩니다.

---

## 2. Vercel 배포 (GitHub 없이, 로컬에서 바로)

GitHub 안 써도 됩니다. **지금 쓰는 폴더에서 터미널로 한 번에** 올릴 수 있어요.

1. [vercel.com](https://vercel.com) 가입만 해 두기.

2. 터미널에서 프로젝트 폴더로 간 뒤:

```bash
cd "라이프플래너 1"
npx vercel
```

3. 처음이면 로그인하라고 나오면 이메일로 로그인.

4. 질문 나오면:
   - **Set up and deploy?** → Yes
   - **Which scope?** → 본인 계정 선택
   - **Link to existing project?** → No
   - **Project name?** → 그냥 Enter (라이프플래너 1 그대로 써도 됨)
   - **In which directory is your code?** → `./` 그대로 Enter

5. 배포 끝나면 **Preview URL** 이 나옵니다 (예: `https://라이프플래너-1-xxx.vercel.app`).

6. Supabase 쓰려면 **환경 변수** 넣어야 합니다. 같은 터미널에서:

```bash
npx vercel env add VITE_SUPABASE_URL
```
→ 프롬프트에 Supabase URL 붙여넣기.

```bash
npx vercel env add VITE_SUPABASE_ANON_KEY
```
→ 프롬프트에 anon key 붙여넣기.

7. 환경 변수 반영해서 다시 배포:

```bash
npx vercel --prod
```

이제 나온 주소로 접속하면 배포된 앱이고, 데이터는 Supabase에 그대로 저장됩니다.

**나중에 수정했으면** 같은 폴더에서 `npx vercel --prod` 한 번 더 실행하면 새 버전으로 배포됩니다. GitHub 없이 계속 이렇게 올리면 됩니다.

---

## 요약

| 구분 | 사용 |
|------|------|
| 데이터 저장 | Supabase (백엔드 서버 없음) |
| 배포 | Vercel (로컬에서 `npx vercel` 만 하면 됨, GitHub 불필요) |
| 난이도 | Supabase 설정 한 번 + 터미널에서 vercel 몇 번 |
